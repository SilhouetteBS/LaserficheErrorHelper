# Code of Conduct

This project follows a simple standard: be useful, factual, and respectful.

## Expected Behavior

- Focus discussion on evidence, product context, and reproducible symptoms.
- Assume good intent when someone reports a correction.
- Explain uncertainty clearly.
- Respect that administrators may be troubleshooting production incidents.

## Unacceptable Behavior

- Harassment, threats, insults, or personal attacks.
- Posting private customer data, credentials, license files, logs with secrets, or repository content.
- Presenting unsupported fixes as confirmed.
- Encouraging unsafe production changes without validation or backup guidance.

## Reporting

Use the repository issue tracker for project behavior concerns, or contact the project maintainers privately if the report includes sensitive information.
